Расширение функциональности и интеграция с базой знаний.

Задание: Реализовать интерфейс управления, позволяющий изменять количество ферзей
(агентов). Подключить базу данных (базу знаний), в которой для каждого агента
фиксируется стоимость его текущей позиции (состояния) и цветовая метка (один из
трех цветов для каждой позиции). Модифицировать логику поиска согласованных
состояний: теперь агенты должны учитывать не только номер позиции, но и
совпадение цветов состояний. Для каждого найденного решения необходимо
отображать и сохранять итоговую сумму стоимостей состояний. Добавить
возможность сортировки решений по возрастанию или убыванию полной
стоимости.

Реализованные классы:

KnowledgeBase - класс для управления базой знаний клеток (веса и цвета)
KnowledgeBaseDialog - диалоговое окно для просмотра и редактирования базы знаний
ChessBoardWidget - виджет для отображения шахматной доски, ферзей и наложения данных БЗ
QueensSolver - класс для решения задачи о N ферзях с использованием backtracking
SortDialog - диалоговое окно для выбора параметров сортировки решений
MainWindow - главное окно приложения

KnowledgeBase: Хранит для каждой клетки вес (1-100) и цвет (красный/зеленый/синий) в SQLite БД с кэшированием. Обеспечивает загрузку/сохранение в файл, генерацию различных распределений и вычисление статистики.

KnowledgeBaseDialog: Таблица N×N для просмотра всех клеток. Каждая ячейка отображает вес и цвет. Поддерживает редактирование, загрузку/сохранение в файл, генерацию распределений (все зеленые, 2 цвета, 3 цвета) и отображение статистики (средний вес, распределение цветов).

ChessBoardWidget: Отрисовывает шахматную доску с чередующимися клетками, накладывает полупрозрачные цвета из БЗ, отображает веса в верхней половине клеток и ферзей в виде кругов. Поддерживает навигацию по решениям и режим редактирования БЗ по клику.

QueensSolver: Рекурсивный backtracking с отсечениями. Проверяет атаки по горизонтали и диагоналям, а также цветовую совместимость (все ферзи одного цвета). Вычисляет веса решений, поддерживает сортировку и асинхронное выполнение в отдельном потоке.

SortDialog: Простое диалоговое окно с выбором направления сортировки (по возрастанию или убыванию веса).

MainWindow: Координирует все компоненты. Управляет поиском решений в отдельном потоке, навигацией по решениям, анимацией, консолью вывода, редактированием БЗ и сортировкой. Обрабатывает изменения размера доски и пересчет весов при редактировании.
